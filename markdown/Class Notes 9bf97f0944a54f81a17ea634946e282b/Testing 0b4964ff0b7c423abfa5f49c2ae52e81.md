# Test plan

- Quality Team responsible for Quality Assurance.
- Leader role usually doing the test plan
- All things related to management of process
- Schedule dates, features, test cases, who is responsible for what

### Typical components

| Element | Descripción |
| --- | --- |
| Alcance | Se definen escenarios de prueba que serán validados. Qué se probará? |
| Fuera del alcance | Mayor claridad sobre lo que no se cubrirá |
| Supuestos | Todas las condiciones que deben ser ciertas para que se pueda proceder con éxito |
| Calendario | Actividades: Preparación, documentación de casos de prueba, datos de prueba, instalar ambiente, ejecución de pruebas, ciclos de pruebas, fechas de inicio y fin |
| Roles y responsabilidades | Se lista los nombres del equipo QA y sus actividades asignadas |
| Entregables | ¿Qué documentos (artefactos de prueba) se van a producir y en qué momento? |
| Ambiente de pruebas | ¿Cuáles son los requerimientos del ambiente de pruebas? ¿Quién será el responsable del ambiente? ¿Qué procederá en caso de tener problemas? |
| Herramientas | Herramienta de Seguimiento de Defectos (Bug Tracking). Ejemplos: JIRA, Bugzilla, Mantis, etc.  |
| Gestión de defectos | ¿A quién se reportan los errores? ¿Cómo se espera que se reporten (texto, screenshots, etc)? ¿Cuál será la clasificación y prioridad de errores (criterios)? |
| Riesgos | Listar riesgos detectados, clasificar riesgos, plan de mitigación de riesgos |
| Criterio de salida | Definir en qué momento se termina el proceso de pruebas |
